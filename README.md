本仓库旨在通过技术手段优化大语言模型（LLM）的访问环境，解决全球用户在订阅和使用 Gemini 与 Claude 时遇到的各类风控拦截问题。

---

## 🚫 解决“Access Denied / 403”报错

AI 平台对访问流量的检测极其严格，常见的拒绝访问通常由以下原因引起：

1. **机房 IP 污染**：公用云服务器 IP 往往在平台的黑名单中。
2. **浏览器指纹泄露**：Canvas 指纹或时区信息与 IP 归属地不符。
3. **WebRTC 泄露**：真实 IP 通过 WebRTC 协议暴露。

**解决方案建议**：
* 强制关闭浏览器 WebRTC。
* 确保链路往返时延（RTT）稳定在 $68ms$ 左右，避免因高延迟触发风控策略。

---

## 🔒 账号长期稳定运营规范

为了保护您的付费订阅资产，请务必遵守以下规范：

* **环境一致性**：严禁频繁在不同国家节点的 IP 间切换登录。
* **对话频率限制**：避免使用自动化脚本进行高频率、无意义的重复提问，以免触发账号滥用检测。
* **敏感词规避**：遵守平台的 Safety Guidelines，尤其是在进行 API 调用时。

* **Gemini 1.5 Pro**：在处理百万级 Token 长文本时，确保上传链路的带宽充足，防止上传中断导致任务失败。
